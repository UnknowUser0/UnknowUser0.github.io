This guide explains how to configure the NoVoidX plugin.

Configuration Overview


# NoVoidX Configuration
enabled: true
check-update: true

worlds:
  world:
    enabled: true
    void-threshold: -128
    target-world: world_nether
    target-y: 512
    ration: 8

  world_the_end:
    enabled: true
    void-threshold: -64
    target-world: world
    target-y: 512
    ration: 1

blindness-duration: 5
teleport-cooldown: 3
log: true

Explanation

  • enabled: Master switch to enable/disable the plugin.
  • check-update: Auto-check for updates on startup.
  • void-threshold: Y-level threshold for void detection.
  • target-world: Destination world when falling into void.
  • target-y: Y-coordinate to teleport player to.
  • ration: Coordinate ratio between worlds (X/Z scaled).
  • blindness-duration: Blindness duration (in seconds) after teleportation.
  • teleport-cooldown: Time (in seconds) before a player can be teleported again.
  • log: Enable or disable logging to console.
