üåç World-specific Configuration

Each world has its own set of configurations. The plugin will monitor if players fall below a certain Y-level and teleport them to a designated world or location.

Example: Overworld (world)


world:
  enabled: true
  void-threshold: -128
  target-world: world_nether
  target-y: 512
  ration: 8
  • enabled: Activates NoVoidX for this world.
  • void-threshold: If a player‚Äôs Y-coordinate goes below -128, they will be teleported.
  • target-world: The destination world, here it's world_nether.
  • target-y: The Y-coordinate where the player will be teleported in the target world.
  • ration: (Likely a typo, should be ratio) ‚Äì The multiplier used to convert the player‚Äôs X/Z coordinates between worlds.

Example: The End (world_the_end)

  enabled: true
  void-threshold: -64
  target-world: world
  target-y: 512
  ration: 1

This configuration teleports players falling below Y -64 in The End to the Overworld at Y 512, maintaining the same X/Z location.

üï∂ Teleportation Effects

  • Description: Prevents a player from being teleported more than once within 3 seconds.

üìù Logging

log: true
  • Description: Enables logging for when players are teleported by NoVoidX. Useful for monitoring and debugging.

üìå Notes

  • Make sure all world names (target-world, etc.) match your actual world folder names.
  • A value like ration: 8 means that if the player is at X/Z 80 in the Overworld, they will be teleported to X/Z 10 in the Nether (80 / 8).
  • Always reload or restart the server after changing the config.
